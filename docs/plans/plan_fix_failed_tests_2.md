# Plan: Fix Failed Solidity Frontend Compilation Tests (Round 2)

## Background

After fixing the ABI JSON test failures (round 1), approximately 40 integration tests still fail. These tests auto-generated by `build.rs` call `test_compiling_solidity_file_standalone` which:
1. Parses Solidity → JSON AST → internal AST
2. Exports internal AST back to Solidity, re-compiles with `solc` (TEST_PARSING_AST)
3. Runs normalization passes (`run_passes`), exports, re-compiles (TEST_NORMALIZING_AST)

Failures fall into distinct root-cause categories analyzed below.

---

## Error Categories & Root Causes

### Category 1: Yul AST Parsing Errors
**Tests:** `memory_guard_test_*`, some `semantic_tests`
**Error:** `"Try statement: parameters not found"`, `"type" field missing` in Yul JSON nodes
**Root Cause:** The JSON AST parser in `ast_parser.rs` expects certain fields (`type`, `parameters`) that may be absent in some Solidity versions or Yul constructs. The parser fails with `error!` instead of gracefully handling `None`.

### Category 2: Normalization Pipeline — `override` with Stale Names
**Tests:** Tests with inheritance (`super`, `override(A, B)`)
**Error:** `solc` rejects re-exported code like `override(C_0)` because `C_0` is a renamed contract name not recognized by `solc`.
**Root Cause:** `strip_specifiers.rs` strips override specifiers **after** `rename_defs`, but `rename_callees` re-inserts renamed contract names into `Overriding::Some(...)`. Since `run_passes` doesn't call `resolve_inheritance`, the re-exported override lists contain stale indexed names.

### Category 3: Normalization Pipeline — `virtual` Not Stripped on Bodyless Functions
**Tests:** Tests with abstract contracts/interfaces
**Error:** `solc` rejects functions missing `virtual` or `override` after normalization strips them.
**Root Cause:** `strip_specifiers.rs` strips `virtual` only when `func.body.is_some()`. After `resolve_inheritance` removes bodyless virtual functions, there shouldn't be remaining issues. However, `run_passes` currently does **not** call `resolve_inheritance`, so abstract virtual functions without bodies get exported with `virtual` stripped but without being removed.

### Category 4: `Overriding` Display — Invalid Solidity Syntax
**Tests:** Tests with `override(A, B)` specifiers
**Error:** `override(ContractName_0, ContractName_1)` is not valid Solidity.
**Root Cause:** The `Display` impl for `Overriding::Some` at [defs.rs:536](file:///home/trung/Workspace/sbip/verazt/crates/frontend/src/solidity/ast/defs.rs#L536-L548) prints `Name` with its index suffix. After renaming, these indexed names produce invalid override specifiers.

### Category 5: Missing `resolve_inheritance` in `run_passes`
**Tests:** All tests with contract inheritance
**Error:** Various `solc` errors about undeclared identifiers, missing function implementations
**Root Cause:** [lower.rs:30-51](file:///home/trung/Workspace/sbip/verazt/crates/frontend/src/solidity/lowering/lower.rs#L30-L51) — `run_passes` doesn't include `resolve_inheritance`. The `super` keyword remains unresolved, and functions marked `abstract/virtual` aren't handled.

### Category 6: `VarDecl` Display — Duplicate Data Location
**Tests:** Multiple tests with `memory`, `storage`, `calldata` parameters
**Error:** `solc` sees `string memory memory x` (duplicate data location).
**Root Cause:** Addressed in round 1 for ABI JSON tests. May still be present in some edge cases during normalization export.

### Category 7: Missing `using for *` Elimination
**Tests:** Tests with `using X for *` directives
**Error:** After normalization, member-access calls like `x.f()` are not rewritten to `X.f(x)`.
**Root Cause:** `eliminate_using.rs` is incomplete — it removes `using` directives but doesn't rewrite member-access calls (has `// FIXME` comments). `run_passes` doesn't call `eliminate_using_directives`.

### Category 8: Test Input Issues
**Tests:** Some test `.sol` files have invalid Solidity
**Error:** Missing SPDX license, missing pragma, or code that `solc 0.8.19` rejects
**Root Cause:** Some libsolidity test files are designed to test specific `solc` error conditions or use features specific to certain Solidity versions. `build.rs` already filters `is_error_test` files but some edge cases slip through.

### Category 9: `TryStatement` Parsing — Missing `parameters` Field
**Tests:** Tests using `try/catch`
**Error:** `"Try statement: parameters not found"` at [ast_parser.rs:1067-1072](file:///home/trung/Workspace/sbip/verazt/crates/frontend/src/solidity/parsing/json_ast_parser/ast_parser.rs#L1067-L1072)
**Root Cause:** Some `solc` versions emit a `TryCatchClause` without a `parameters` field when there are no return parameters. The parser currently treats this as a hard error.
