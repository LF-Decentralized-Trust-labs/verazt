# Prerequisites

## For Solidity smart contracts

- We use `solc-select` to switch between Solidity compiler versions.

```bash
pip3 install solc-select
```

## For Vyper smart contracts

- We use `vyper-select` to switch between Vyper compiler versions.

```bash
pip3 install vyper-select
```

# Usage

## Analyzing Solidity contracts

```bash
verazt analyze <path/to/contract.sol>
```

## Analyzing Vyper contracts

```bash
verazt analyze <path/to/contract.vy>
```

# Testing

## Running all tests

```bash
cargo test --no-fail-fast
```

> **Note:** The `--no-fail-fast` flag ensures that all test targets (lib, integration,
> dataset) are run even if earlier targets have failures.

## Running dataset tests

The **frontend** crate includes auto-generated tests for the **libsolidity** and
**smartbugs-curated** datasets. Each `.sol` file gets its own `#[test]` function,
generated by `build.rs` at compile time.

```bash
# Run libsolidity dataset tests
cargo test -p frontend --test compile_libsolidity

# Run smartbugs-curated dataset tests
cargo test -p frontend --test compile_smartbugs
```

# Notes

- For Solidity: only support Solidity version >= 0.4.12
